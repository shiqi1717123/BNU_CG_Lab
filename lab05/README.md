# Lab05: Whitted-Style 光线追踪

## 实验概述

这是一个很直观而且有意思的实验，实现了一个基于Whitted Style模型的简单光线追踪器。基础部分隐式定义了一个无限棋盘格地面、红色漫反射球和银色镜面球，并通过迭代循环完成反射光线追踪、硬阴影检测和实时交互。

选做内容如下：

- 选做1：折射与玻璃材质。将左侧红球改为玻璃球，使用斯涅尔定律计算折射方向，并在发生全反射时切换为反射方向。
- 选做2：抗锯齿。保留基础场景材质，在每个像素内发射多条抖动后的主射线，平均颜色结果。

## 场景与材质

基础实验包含三个几何体：

- 无限大地面：位于 `y = -1.0`，法线为 `(0, 1, 0)`，使用黑白棋盘格纹理，材质为漫反射。
- 红色漫反射球：位于 `(-1.5, 0.0, 0.0)`，半径为 `1.0`。
- 银色镜面球：位于 `(1.5, 0.0, 0.0)`，半径为 `1.0`，材质为理想镜面反射。

选做1，左侧红色漫反射球被替换为玻璃球。玻璃球仍使用相同的位置和半径，但材质行为改为折射。

选做2，场景材质保持基础版不变，即左侧仍为红色漫反射球，右侧仍为银色镜面球；只改变像素采样方式来减少锯齿。

## 交互说明


- `Light X`、`Light Y`、`Light Z`：调整点光源位置，观察硬阴影实时移动。
- `Max Bounces`：最大弹射次数，范围为 `1 ~ 5`，默认值为 `3`。

选做1：

- `Glass IOR`：调整玻璃折射率，默认值为 `1.5`。

选做2：

- `Samples Per Pixel`：调整每个像素的采样数量，范围为 `1 ~ 8`，默认值为 `4`。

`Samples Per Pixel = 1` 时，每个像素只从中心发出一条主射线，边缘锯齿最明显。采样数增大后，每个像素内会发出多条带有稳定随机偏移的主射线，并将颜色平均，因此球体轮廓、棋盘边缘和反射边缘会更平滑。

## 镜面与玻璃材质区别

在选做1里面我们可以看到两个小球材质的区别，最直观的就是玻璃小球成像为倒影，但镜面球是镜像。

镜面材质只进行反射。光线命中镜面球后，会按照反射公式得到新的方向，继续追踪下一次命中结果。因此镜面球的颜色主要来自反射方向上看到的场景。

玻璃材质主要进行折射（也会反射）。光线命中玻璃球后，需要根据斯涅尔定律计算透射方向。一次完整穿过玻璃通常需要两次表面相交：先进入玻璃，再从另一侧离开玻璃。如果从玻璃内部射向外部时无法满足折射条件，就会发生全反射，此时程序改用反射方向继续追踪。

可以把理解为：

```text
镜面：相机-镜面表面-反射方向上的场景
玻璃：相机-玻璃前表面-玻璃内部-玻璃后表面-外部场景
```

所以镜面通常只需要较少弹射次数（2次）就能看到环境，而玻璃需要更多弹射次数（至少3次）才能真正表现出透明效果。

## 浅析Max Bounces的变化，观察现象

`Max Bounces` 控制每条光线最多传播多少段路径。它对镜面和玻璃的影响非常明显：

- `Max Bounces = 1`：光线只到达第一层表面。镜面球和玻璃球都还没有机会采样反射或折射后的场景，所以两个球看起来都是黑的。
- `Max Bounces = 2`：镜面球可以反射到环境或地面，因此变亮；玻璃球通常只完成进入和离开玻璃，还没有继续命中外部场景，所以玻璃球或玻璃球倒影仍可能偏黑。
- `Max Bounces = 3`：直接看到的玻璃球已经能够穿出玻璃并采样外部场景，因此两球都开始发亮；但镜面中的玻璃球路径更长，倒影仍可能是黑的。
- `Max Bounces = 4和5`：镜面里看到玻璃球的路径也足够完整，例如 `相机-镜面-玻璃前表面-玻璃后表面-场景`，所以镜面球中开始出现亮的玻璃球倒影。


总结来说，镜面反射通常 `2` 次弹射就能看到内容；玻璃穿透通常至少需要 `3` 次；如果是镜面中再看到明亮的玻璃球倒影，通常至少需要 `4` 次。


## 抗锯齿观察现象

基础版每个像素只发出一条穿过像素中心的主射线，因此物体轮廓和棋盘格边缘会出现明显台阶状锯齿。

选做2加入了 MSAA 思路：每个像素最多发射 `8` 条主射线。这些射线的起点仍来自相机，但会穿过当前像素内部的不同随机位置。程序对多条射线得到的颜色取平均，得到该像素的最终颜色。

调节 `Samples Per Pixel` 可以观察到：

- `1`：速度最快，但边缘最硬，锯齿明显。
- `2 ~ 4`：球体边缘和棋盘远处纹理明显变顺，实时性能仍较好。
- `8`：边缘最平滑，但计算量约为单采样的 8 倍，帧率会下降。

为了避免画面每帧闪烁，代码使用基于像素坐标和采样编号的稳定伪随机抖动，而不是每帧重新生成完全不同的随机数。

## 实现要点

### 1. 迭代式光线追踪

GPU不适合直接使用递归，因此程序在每个像素中使用固定上限的 `for` 循环来模拟递归光线追踪。每条光线维护两个变量：

- `throughput`：记录光线经过反射或折射后的衰减系数。
- `final_color`：记录最终累积颜色。

当光线命中漫反射表面时，程序计算 Phong 光照并将结果乘以 `throughput` 加入 `final_color`，然后结束循环。命中镜面或玻璃表面时，只更新射线起点、方向和 `throughput`，继续下一次循环。

### 2. 硬阴影

漫反射表面着色时，会从交点向光源发射一条阴影射线。如果该射线在到达光源前命中其他非透明物体，就只保留环境光，形成硬阴影。

为了避免 Shadow Acne，自交问题，反射射线、折射射线和阴影射线都会做一个很小的起点偏移：

```text
P_new = P + direction_or_normal * EPSILON
```

其中 `EPSILON = 1e-4`。

### 3. 折射与全反射

选做1中，玻璃球使用折射率 `IOR` 控制折射强度。程序根据当前光线是在进入玻璃还是离开玻璃，选择空气和玻璃的折射率比，并计算折射方向。

当计算得到的 `sin^2(theta_t) > 1` 时，说明折射光线不存在，此时发生全反射，程序改为使用反射方向继续传播。

### 4. 多重采样抗锯齿

选做2中，MSAA 版本将单条主射线扩展为多条主射线。对于每个像素，程序根据 `Samples Per Pixel` 循环采样：

```text
pixel_color = (sample_0 + sample_1 + ... + sample_n) / n
```

每条样本射线都完整执行一次反射和阴影计算。这样可以在物体边缘处混合前景和背景颜色，减少像素级跳变，从而得到更平滑的轮廓。
